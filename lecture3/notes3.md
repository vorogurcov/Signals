# Конспект: Синдромное декодирование и границы параметров

## 1. Постановка задачи декодирования
При передаче по двоичному симметричному каналу (ДСК) кодовое слово $c \in C$ искажается вектором ошибки $e$. Принятый вектор $y$:
$$y = c \oplus e$$

**Декодирование по максимуму правдоподобия (MLD):**
Поиск кодового слова $\hat{c}$, минимизирующего расстояние Хэмминга $d(y, \hat{c})$. В $GF(2)$ это эквивалентно поиску вектора ошибки $\hat{e}$ с минимальным весом Хэмминга $w(e)$:
$$\hat{e} = \arg \min_{e: (y \oplus e) \in C} w(e)$$

---

## 2. Смежные классы (Косеты)
Разбиение пространства $F_2^n$ на непересекающиеся подмножества относительно кода $C$.

> **Определение:** Смежным классом кода $C$ по вектору $a \in F_2^n$ называется множество:
> $$a \oplus C = \{ a \oplus c \mid c \in C \}$$

### Свойства смежных классов:
* **Количество классов**: $2^{n-k}$ (индекс подгруппы).
* **Мощность каждого класса**: $|C| = 2^k$ векторов.
* **Лидер смежного класса**: Вектор с минимальным весом Хэмминга в данном классе. Соответствует наиболее вероятной ошибке.

### Стандартная таблица декодирования (Standard Array):
| Лидеры ($e$) | $c_0 = \mathbf{0}$ | $c_1$ | ... | $c_{2^k-1}$ |
| :--- | :--- | :--- | :--- | :--- |
| **$e_0 = \mathbf{0}$** | $c_0$ | $c_1$ | ... | $c_{2^k-1}$ |
| **$e_1$** | $e_1 \oplus c_0$ | $e_1 \oplus c_1$ | ... | $e_1 \oplus c_{2^k-1}$ |
| **$e_j$** | $e_j \oplus c_0$ | $e_j \oplus c_1$ | ... | $e_j \oplus c_{2^k-1}$ |

---

## 3. Синдромное декодирование
Метод, позволяющий сократить объем хранимых данных с $2^n$ до $2^{r}$ записей, где $r = n-k$.

> **Определение:** Синдромом принятого вектора $y$ называется вектор $S$ длины $r$:
> $$S = y \cdot H^T$$

### Свойства синдрома:
1. **Инвариантность**: Синдром зависит только от ошибки: $S = (c \oplus e)H^T = e \cdot H^T$.
2. **Биекция**: Существует взаимно однозначное соответствие между синдромами и лидерами смежных классов.

### Алгоритм декодирования:
1. **Offline**: Создание таблицы соответствия «Синдром $\to$ Лидер».
2. **Online**: 
    * Вычисление $S = y \cdot H^T$.
    * Поиск $e$ по таблице синдромов.
    * Исправление: $\hat{c} = y \oplus e$.

---

## 4. Геометрическая интерпретация
Декодирование — это процесс определения принадлежности вектора $y$ «сфере влияния» конкретного кодового слова.

* **Сфера Хэмминга** радиуса $t$: множество векторов на расстоянии $d \le t$ от центра.
* **Объем сферы $V(n, t)$**:
$$V(n, t) = \sum_{i=0}^{t} \binom{n}{i}$$



---

## 5. Границы параметров кодов

### 5.1. Граница Хэмминга (Верхняя / Плотностной предел)
Устанавливает максимально возможное число кодовых слов $M$ для заданного $n$ и $t$:
$$M \cdot \sum_{i=0}^{t} \binom{n}{i} \le 2^n$$

**Совершенные коды**: Коды, достигающие этой границы (сферы заполняют пространство без пробелов).
* Коды Хэмминга ($t=1$).
* Код Голея (23,12,7) ($t=3$). Проверка: $2^{12} \cdot 2048 = 2^{23}$.

### 5.2. Граница Варшамова–Гилберта (Нижняя / Экзистенциальная)
Гарантирует существование линейного $(n, k)$-кода с расстоянием $d$, если выполняется условие:
$$2^{n-k} > \sum_{i=0}^{d-2} \binom{n-1}{i}$$

---

## 6. Резюме: Сравнение границ
| Граница | Тип | Смысл |
| :--- | :--- | :--- |
| **Хэмминга** | Верхняя | Кодов лучше этого предела **не существует**. |
| **Варшамова–Гилберта** | Нижняя | Код с такими параметрами **гарантированно существует**. |
